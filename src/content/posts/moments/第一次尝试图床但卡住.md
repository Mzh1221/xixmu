在搭建这个 firefly 博客的时候我担心这种博客要不要弄上图床

在写[电话线发圈真不错](https://162351.xyz/posts/coil-hair-tie)的时候想传几张手机自拍，但是发现现在手机照片太大了，随便就是十几 MB，直接放 src 里也太大了，遂想要弄个图床。

我本来是跟着 [CloudFlare ImgBed](https://cfbed.sanyue.de/) 这个项目弄图床。

结果我网站搭好了 [Sanyue ImgHub](https://imgbed.162351.xyz/) 却发现卡在了存储渠道上了。

| 渠道类型          | 优点            | 限制                    |
| ------------- | ------------- | --------------------- |
| Telegram Bot  | 完全免费、无限容量     | 大于20MB文件需分片存储         |
| Cloudflare R2 | 无文件大小限制、企业级性能 | 超出10G免费额度后收费，需要绑定支付方式 |
| S3 兼容存储       | 选择多样、价格灵活     | 根据服务商定价               |
| Discord       | 完全免费、简单易用     | 大于10MB文件需分片存储         |
| HuggingFace   | 完全免费、支持大文件直传  | 需要 HuggingFace 账号     |
| WebDAV        | 兼容性广、支持自建存储   | 根据服务商定价               |

弄 Cloudflare R2 时发现要绑银行卡，但是我直接绑国内卡不行，看到有说要绑 PayPal，但是好像说会给银行卡预留电话发短信还是什么，我那张卡在老家，我在学校。

寻求别的方式，好像 S3 兼容存储可以用，七牛云和又拍云。

|服务商|地区|特点|
|---|---|---|
|Amazon S3|全球|最原始的 S3 服务|
|七牛云|中国|国内访问速度快|
|又拍云|中国|CDN 加速|
|Backblaze B2|美国|价格便宜|
|Cloudflare R2|全球|与 Cloudflare 生态集成|
|MinIO|全球|开源 S3 兼容存储|

不过这样是收费的，于是我暂时搁置图床搭建。

先用着 [iLoveIMG | 图像文件在线编辑工具](https://www.iloveimg.com/zh-cn) 压缩图片，[路过图床 - 免费图片上传, 专业图片外链, 免费公共图床](https://imgchr.com/) 来当图床。

我暂时写动态没有很大的图片需求，而且生活图片在手机里，直接在手机浏览器压缩上传，直接去电脑去路过图床复制链接，这个流程还算可以接受。

不过我之后要是写教程配图的话还是得继续推进一下这个图床的工作流。
